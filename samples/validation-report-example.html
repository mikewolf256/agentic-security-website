Sample validation report (structure)

This is a simplified example of what “validated” looks like. The goal is clarity: reproduce quickly, understand impact, and know the confidence level.

Title

Potential auth bypass (candidate)

Status

Not confirmed (false positive)

Confidence

High

Severity

N/A (candidate disproven)

Summary

Scanner flagged an authentication bypass based on a 200 response and “homepage-looking” content. Validation shows the endpoint returns the same homepage response for payload and control requests; no authentication logic is exercised.

Reproduction (example)

  1. Send a baseline request (control).
  2. Send the candidate payload request.
  3. Compare response markers (status, size, keywords).

Evidence (example excerpts)

CONTROL
POST /login
...

HTTP/1.1 200 OK
Content-Length: 412833
...
<title>Home</title>

PAYLOAD
POST /login
...

HTTP/1.1 200 OK
Content-Length: 412833
...
<title>Home</title>

Conclusion: response-shape is identical; candidate is not exploitable.

What “validated” means on our reports

  • Confirmed findings include reproduction steps + evidence + remediation guidance.
  • Unconfirmed candidates are clearly labeled with confidence and what’s missing to confirm.
  • False positives are closed with a short explanation and proof.
